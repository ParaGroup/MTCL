RequestPool &  r

pure virtual

Post a non-blocking receive of one message into buff.

The parameter size is the capacity of the user-provided buffer buff. The operation completes when a full message has been received (and copied) or when the peer closes the connection (EOS).

Completion semantics:

  • The associated request becomes "complete" when the receive finishes.
  • The actual number of bytes received can be retrieved via Request::count() after completion.
Parameters
buffDestination buffer.
sizeCapacity of buff in bytes.
rRequest object (or pool) used to track completion.
Returns
Returns 0 on successful posting of the operation, or -1 on immediate error (with errno set). If the incoming message is larger than size, the request completes with error EMSGSIZE (the message is drained).

Implemented in MTCL::HandleMQTT, MTCL::CollectiveContext, MTCL::HandleMPI, MTCL::HandleTCP, and MTCL::HandleUCX.

  • After completion, the user may safely reuse or free buff.

const bool  blocking = true 
)
pure virtual

Check whether an incoming message is available and obtain its size.

If a message is available, this call sets size to the message length in bytes. Depending on the backend, the size information may be cached so that a subsequent receive() can proceed without re-probing.

Parameters
[out]sizeMessage size in bytes when a message is available.
[in]blockingIf true, block until a message is available or the peer closes. If false, return immediately and set errno to EWOULDBLOCK (or EAGAIN) if no message is currently available.
Returns
Returns a positive value on success (implementation-defined, typically sizeof(size_t)). Returns 0 if the peer has closed the connection (EOS). Returns -1 on error and sets errno accordingly.

Implemented in MTCL::CollectiveContext, MTCL::HandleMPI, MTCL::HandleMPIP2P, MTCL::HandleMQTT, MTCL::HandleSHM, MTCL::HandleTCP, and MTCL::HandleUCX.

virtual ssize_t MTCL::CommunicationHandle::receive ( void *  buff,
size_t  size 
)
pure virtual

Receive one message from the remote end into buff.

The parameter size is the capacity of the user-provided buffer buff. The implementation may internally read (or have already read via probe()) the message size, then copies at most size bytes into buff.

Parameters
buffDestination buffer.
sizeCapacity of buff in bytes.
Returns
On success, returns the number of bytes actually received and copied into buff (0 <= n <= size). Returns 0 only to signal end-of-stream (the peer closed the connection / EOS). Returns -1 on error and sets errno. If the incoming message is larger than size, returns -1 and sets errno to EMSGSIZE (the message is not consumed, and thus may be retried).

Implemented in MTCL::CollectiveContext, MTCL::HandleMPI, MTCL::HandleMPIP2P, MTCL::HandleMQTT, MTCL::HandleSHM, MTCL::HandleTCP, and MTCL::HandleUCX.
